## 🧠 业务核心逻辑 (Solution Architecture)

作为解决方案经理，我将该风控方案拆解为以下 **6 步原子化流程**：

### 1. 触发阶段 (Trigger)
医生在处方界面输入药物（二甲双胍）及剂量，并点击“提交”。此时系统处于“准提交”状态。

### 2. 检索阶段 (Retrieval)
AI 安全引擎通过 API 实时检索患者最新的临床指标：**eGFR (肾小球滤过率)**。

### 3. 判定阶段 (Logic Judge)
根据内置专家共识协议进行逻辑分算：
* **绝对禁忌**：$eGFR < 30$ mL/min/1.73m²
* **相对风险**：$30 \le eGFR < 45$ mL/min/1.73m²

### 4. 拦截动作 (Atomic Action) —— **核心容错设计**
* **硬拦截 (Hard Block)**：若触发绝对禁忌，系统执行**原子性拦截**，直接冻结“提交”数据库的操作，确保错误数据零入库。

### 5. 交互反馈 (UI/UX Response)
系统弹出高亮预警窗：
* **标题**：【严重用药风险拦截】
* **内容**：明确告知风险原因（乳酸酸中毒风险）及指标依据。
* **出口**：仅允许“撤回”或“修改”，不提供“忽略”选项。

### 6. 闭环管理 (Closed-loop Admin)
系统自动生成 `Audit Log`（审计日志），并将该异常行为同步推送至**药剂师审核端**，作为后续科室质控依据。
